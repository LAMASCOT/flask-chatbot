PARAMETRAGE POUR LA RECUPERATION DES COMPTES

Introduction

Ce paramétrage permet de définir les modalités d’accès aux comptes des clients dans le système bancaire. Il oriente le comportement de l’application quant à la source de vérité des informations de comptes : soit en local (base de données propre à l’application), soit via des appels API vers le Core Banking System (CBS).

Ces paramètres sont essentiels pour garantir l’intégration fluide et sécurisée entre l’interface utilisateur et le système bancaire central, en tenant compte des choix d’architecture de chaque banque cliente.

Accès local à la base des comptes

  • Nom du paramètre : LOCAL_BASE_COMPTE

  • Valeur par défaut : OUI

  • Libellé : LOCAL BASE COMPTE

  • Type : CODE_PARAMS

Description :
Ce paramètre indique si l’application doit accéder aux comptes des clients à partir d’une base de données locale (OUI) ou non (NON). Lorsqu’il est activé, l’application interroge une base interne pour récupérer les informations des comptes. Cela peut être utile en mode déconnecté ou en cas d’absence d’intégration directe avec le système bancaire central.

Accès aux comptes via une API (Core Banking)

  • Nom du paramètre : FETCH_COMPTE_SERVICE

  • Valeur par défaut : OUI

  • Libellé : FETCH_COMPTE_SERVICE

  • Type : CODE_PARAMS

Description :
Ce paramètre contrôle l’activation de la récupération des comptes via une API exposée par le Core Banking System. Lorsqu’il est à OUI, l’application interagit directement avec le système bancaire central pour récupérer les comptes à jour et validés, assurant une synchronisation en temps réel. Cela garantit une meilleure cohérence des données, en particulier pour les opérations sensibles ou complexes.

Système bancaire principal

  • Nom du paramètre : CORE_SYSTEM

  • Valeur par défaut : AMPLITUDE

  • Libellé : Core Banking system

  • Type : CODE_PARAMS

Description :
Ce paramètre spécifie le nom du Core Banking System utilisé par la banque. Ici, la valeur AMPLITUDE indique que l’infrastructure bancaire principale repose sur le système AMPLITUDE. Cette information peut être exploitée par l’application pour adapter dynamiquement certains comportements ou intégrations selon le CBS en place.
