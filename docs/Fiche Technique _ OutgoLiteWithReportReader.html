Introduction

Description : La tache OutgoLiteWithReportReader est conçue pour lire et traiter les fichiers plats contenant des informations sur les transactions sortantes pour les entreprises, en mettant à jour les enregistrements dans la base de données et en générant des rapports.

Package

Emplacement : sas.action.readers.corporates

Description Fonctionnelle

Déclenchement : Activée lors de la détection de nouveaux fichiers plats dans le répertoire configuré.

Traitement :

Lit le fichier pour extraire les en-têtes de remise et les opérations associées.

Met à jour ou insère les enregistrements de transactions dans la base de données.

Gère la génération de rapports pour les transactions traitées.

Résultat : Les transactions sont traitées et les rapports sont générés.

En cas d’erreur : Les erreurs de traitement sont consignées dans les journaux pour un suivi ultérieur.

Opérations Clés

Lecture des Fichiers : Utilise BufferedReader pour lire les lignes du fichier et extraire les données.

Mise à Jour des Enregistrements :

Utilise DataBaseJPA pour insérer ou mettre à jour les enregistrements de transactions dans la base de données.

Gère les différents types de transactions (virements, prélèvements, chèques, etc.) en fonction des codes d'opération.

Génération de Rapports :

Utilise GenerateReportingOrdersCorporates pour générer des rapports basés sur les transactions traitées.

Paramètres du Composant

Répertoires : Utilise des objets Repertoires pour gérer les répertoires de fichiers.

Paramètres de Configuration :

SEND_TICKET, CHEMIN_LOGO pour la gestion des notifications et des logos.

Interactions avec la Base de Données

Requêtes SQL : Utilise des requêtes SQL pour récupérer et mettre à jour les enregistrements de comptes et de transactions.

Gestion des Transactions : Utilise DataBaseJPA pour gérer les transactions de base de données.

Paramètres Utilisés et Valeurs Possibles

SEND_TICKET : Indique si un ticket doit être envoyé.

CHEMIN_LOGO : Chemin du logo utilisé dans les emails.

Remarques

Implémentation Future : La méthode treatFile est partiellement implémentée et peut nécessiter des ajustements pour gérer de nouveaux types de transactions ou de nouvelles exigences de traitement.
