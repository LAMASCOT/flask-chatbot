Introduction

La tache ICRCheckReader est conçu pour traiter automatiquement les chèques scannés accompagnés de leur bordereau de remise, en utilisant les capacités de reconnaissance ICR (Intelligent Character Recognition). Il est déclenché via le Gestionnaire de Processus dès que des fichiers sont détectés dans le répertoire associé à une machine de scan. Ce traitement est particulièrement adapté pour les institutions bancaires cherchant à automatiser le traitement des remises tout en garantissant la qualité des données extraites (numéros, montants, RIB, etc.). Le composant permet une intégration fluide avec la base de données SmartACCESS et alimente les étapes de validation et de compensation.

 

Description fonctionnelle

Déclenchement: Le composant détecte les fichiers déposés dans le dossier configuré.

Traitement: Il analyse les images, reconnaît les caractères (montant, numéro de compte, etc.), et vérifie la cohérence avec les bordereaux.

Résultat: Les données préparées sont alimentées dans la base SmartACCESS, et des journaux d’exécution sont générés et visibles dans le Gestionnaire de Processus.

En cas d’erreur: Les anomalies sont inscrites dans les journaux avec des codes couleurs (succès, avertissement, erreur) pour faciliter le suivi.

 

Opérations Clés

Gestion des Fichiers : La classe gère les fichiers représentant les chèques, en s'assurant qu'ils sont complets et non dupliqués.

Interaction avec la Base de Données : Elle récupère et stocke des informations sur les chèques et les remises dans la base de données.

Traitement d'Images : La classe peut redimensionner les images et les déplacer vers des répertoires spécifiques.

 

Paramètres du composant

RESIZE_THE_IMAGE :

Valeurs possibles : "OUI", "NON"

Comportement :

Si la valeur est "OUI", les images seront redimensionnées à une largeur de 1200 pixels lors de leur traitement.

Si la valeur est "NON", les images seront copiées sans redimensionnement.

IMG_ALLER_FOLDER :

Valeur possible : Chemin d'accès, par exemple, "d:\\image\\aller"

Comportement : Définit le répertoire de destination pour les images traitées.

RVDSTRONGCHECK :

Valeurs possibles : "0", "1", "2"

Comportement :

"0" : Les fichiers déjà scannés ne sont pas déplacés ou supprimés.

"1" : Les fichiers déjà scannés sont déplacés vers un répertoire de "junk" spécifié par le paramètre JUNK_FOLDER.

"2" : Les fichiers déjà scannés sont supprimés.

JUNK_FOLDER :

Valeur possible : Chemin d'accès, par exemple, "C:/JunkFolder"

Comportement : Spécifie le répertoire où les fichiers déjà scannés sont déplacés si RVDSTRONGCHECK est "1".

STRONG_CHECK_CMC7 :

Valeurs possibles : "0", "1"

Comportement :

"0" : Pas de vérification stricte de la longueur de la ligne CMC7.

"1" : Vérifie que la ligne CMC7 a une longueur minimale définie par CMC7_MIN_LENGTH.

CMC7_MIN_LENGTH :

Valeur possible : Un entier, par exemple, 30

Comportement : Définit la longueur minimale requise pour la ligne CMC7 si STRONG_CHECK_CMC7 est "1".

CETAREMSTO :

Valeur possible : Un code d'état, par exemple, 10

Comportement : Utilisé pour définir l'état d'une remise lors de sa création.
