Introduction

Description : La tache MiseEnPortefeuilleChequesAllerImalWriter est une extension de FlatFileWriter utilisée pour traiter et envoyer des chèques aller vers le système IMAL. Elle gère la mise en portefeuille des chèques en vérifiant les comptes internes et en mettant à jour les informations des chèques dans la base de données.

Package

Emplacement : sas.action.writers.forimal

Description Fonctionnelle

Objectif : Traiter et envoyer des chèques aller vers IMAL en respectant le schéma comptable défini.

Traitement :

Vérifie la configuration des comptes internes nécessaires pour le traitement.

Récupère les chèques aller disponibles pour le traitement.

Met à jour les informations des chèques et les envoie vers IMAL.

Gère les verrous pour éviter les traitements concurrents.

Méthodes

MiseEnPortefeuilleChequesAllerImalWriter() : Constructeur par défaut.

execute() : Exécute le processus de mise en portefeuille des chèques aller vers IMAL.

fetchCheques() : Récupère les chèques aller disponibles pour le traitement à partir de la base de données.

Attributs

dataBaseJPA : Instance de DataBaseJPA pour interagir avec la base de données.

parameters : Map pour stocker les paramètres de requête.

LOG : Logger pour enregistrer les événements et les erreurs.

valeurEncaissementNonDisponible : Compte pour les valeurs à l'encaissement non disponibles.

cptClienteleExigibleEncaissement : Compte pour la clientèle exigible après encaissement.

cptRecouvrement : Compte de recouvrement.

cptContrePartieClient : Compte de contrepartie pour le recouvrement client.

cptFraisBCEAO : Compte pour les frais BCEAO.

Paramètres Utilisés

CPT_ENCAISSEMENT_NON_DISPONIBLE : Compte pour les valeurs à l'encaissement non disponibles.

CPT_CLIENTELE_EXIGIBLE_ENCAISSEMENT : Compte pour la clientèle exigible après encaissement.

CPT_RECOUVREMENT : Compte de recouvrement.

CPT_CONTREPARTIE_CLIENT : Compte de contrepartie pour le recouvrement client.

CPT_FRAIS_BCEAO : Compte pour les frais BCEAO.

LOCK_CHK_ALL_PORT_COMPENSE : Paramètre pour gérer le verrouillage du traitement des chèques.

CETAOPEALLICOM1ACC : État des chèques après mise à jour.

Remarques

Gestion des Erreurs : Les erreurs lors du traitement des chèques sont gérées et enregistrées dans les logs.

Extensibilité : La tache peut être étendue pour inclure d'autres traitements ou vérifications spécifiques aux chèques.

Intégration : Conçue pour être utilisée dans des contextes où les chèques doivent être gérés et suivis dans le système IMAL, en respectant les règles comptables définies.
