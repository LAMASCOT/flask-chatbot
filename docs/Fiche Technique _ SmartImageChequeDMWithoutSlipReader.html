Introduction

Description : La tache SmartImageChequeDMWithoutSlipReader est une extension de FlatFileReader utilisée pour lire et traiter les fichiers d'images de chèques scannés sans bordereau. Elle gère la validation des données, le traitement des images, et l'intégration des informations dans le système bancaire.

Description Fonctionnelle

Déclenchement : Activée lors de la détection de nouveaux fichiers d'images de chèques dans le répertoire configuré.

Traitement : Vérifie les données des fichiers scannés, traite les images des chèques, et met à jour les informations dans la base de données.

Résultat : Mise à jour des statuts des chèques et des séquences dans la base de données, avec les images des chèques déplacées vers le répertoire approprié.

En cas d’erreur : Les erreurs sont consignées dans les journaux pour un suivi ultérieur.

Opérations Clés

Gestion des Fichiers : Assure la lecture, la validation, et le déplacement des fichiers d'images de chèques.

Interaction avec la Base de Données : Met à jour les statuts des chèques et des séquences.

Traitement d'Images : Redimensionne et compresse les images des chèques.

Vérification de Conformité : Vérifie si les fichiers ont déjà été traités pour éviter les doublons.

Lecture de Codes DataMatrix : Utilise BarCodeReader pour extraire et traiter les données des codes DataMatrix présents sur les images de chèques. Ces données sont utilisées pour enrichir les informations sur le chèque et vérifier son authenticité.

Paramètres du Composant

IMG_ALLER_FOLDER : Répertoire de destination pour les images traitées.

DT_SNIPPET_X, DT_SNIPPET_Y, DT_SNIPPET_W, DT_SNIPPET_H : Coordonnées et dimensions pour extraire le sous-image contenant le code DataMatrix.

ENDOS_SNIPPET_WIDTH : Largeur du sous-image pour lire le numéro d'endossement.

Méthodes Clés

setRepertoire : Configure le répertoire d'attente pour les fichiers à traiter.

getAlphaSicaCode : Récupère le code alphanumérique SICA pour un code numérique donné.

resizeUsingJavaAlgo : Redimensionne une image en utilisant un algorithme Java.

writeJpeg : Écrit un fichier JPEG en définissant la qualité de compression.

secondReadDataMatrixCode : Lit le code DataMatrix d'une image de chèque.

readNumeroEndos : Lit le numéro d'endossement d'une image de chèque.

treatFile : Traite un fichier d'image de chèque, en gérant le déplacement et la mise à jour des informations dans la base de données.

Cette classe est essentielle pour le traitement automatisé des chèques scannés, garantissant que les informations sont correctement extraites et intégrées dans le système bancaire.
