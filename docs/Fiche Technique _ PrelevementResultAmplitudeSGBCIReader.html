Introduction

Description : La tache PrelevementResultAmplitudeSGBCIReader est conçue pour lire et traiter les fichiers plats contenant des résultats de prélèvements pour SGBCI, en mettant à jour les enregistrements dans la base de données.

Package

Emplacement : sas.action.readers.corporates

Description Fonctionnelle

Déclenchement : Activée lors de la détection de nouveaux fichiers plats dans le répertoire configuré.

Traitement :

Lit le fichier pour extraire les informations de prélèvement.

Met à jour les enregistrements de prélèvements dans la base de données en fonction des résultats de traitement.

Résultat : Les prélèvements sont traités et mis à jour dans la base de données.

En cas d’erreur : Les erreurs de traitement sont consignées dans les journaux pour un suivi ultérieur.

Opérations Clés

Lecture des Fichiers : Utilise BufferedReader pour lire les lignes du fichier et extraire les données.

Mise à Jour des Enregistrements :

Utilise DataBaseJPA pour mettre à jour les enregistrements de prélèvements dans la base de données.

Gère les informations de prélèvement telles que le montant, les comptes bénéficiaires et tireurs, et les détails de l'émetteur.

Met à jour l'état et le motif de rejet des prélèvements en fonction des résultats de traitement.

Paramètres du Composant

Répertoires : Utilise des objets Repertoires pour gérer les répertoires de fichiers.

Paramètres de Configuration :

CODE_BANQUE_SICA3, ORDPRENOUNOR, CETAOPEALLICOM2, CETAOPERETENVSIB pour la gestion des prélèvements.

Interactions avec la Base de Données

Requêtes SQL : Utilise des requêtes SQL pour mettre à jour les enregistrements de prélèvements.

Gestion des Transactions : Utilise DataBaseJPA pour gérer les transactions de base de données.

Paramètres Utilisés et Valeurs Possibles

CODE_BANQUE_SICA3 : Code de la banque utilisé pour les prélèvements.

ORDPRENOUNOR : Type de prélèvement.

CETAOPEALLICOM2 : État des prélèvements acceptés.

CETAOPERETENVSIB : État des prélèvements en attente.

Remarques

Implémentation Future : La méthode treatFile est partiellement implémentée et peut nécessiter des ajustements pour gérer de nouveaux types de prélèvements ou de nouvelles exigences de traitement.
