Introduction

Description : La tache AmplitudeTransfertApiClient est un client API REST utilisé pour interagir avec le service Amplitude pour effectuer des transferts. Elle gère la création et l'envoi de requêtes de transfert, ainsi que le traitement des réponses et des erreurs.

Package

Emplacement : sas.action.writers.corporates.sfd.sica

Description Fonctionnelle

Objectif : Fournir une interface pour interagir avec l'API Amplitude pour les opérations de transfert.

Traitement :

Configure un client REST pour communiquer avec l'API Amplitude.

Envoie des requêtes de transfert et traite les réponses.

Gère les erreurs et les exceptions lors des appels API.

Attributs

dataBaseJPA : DataBaseJPA - Instance pour l'accès à la base de données.

LOG : Logger - Logger pour enregistrer les informations et les erreurs.

createTransactions : WebTarget - Cible pour la création de transactions.

transfertEntries : WebTarget - Cible pour les entrées de transfert.

client : Client - Client REST pour effectuer les appels API.

BASE_URI : String - URI de base pour l'API Amplitude, récupéré à partir des paramètres de configuration.

Méthodes

AmplitudeTransfertApiClient() : Constructeur qui initialise le client REST et configure les cibles API.

createTransfers(TransferApiAmplitudeRequest transferToApis, String apiKey, String etabPartenaire, String codeProduit) : Envoie une requête de transfert à l'API Amplitude et retourne la réponse.

returnMapper() : Retourne un ObjectMapper configuré pour la sérialisation/désérialisation JSON.

close() : Ferme le client REST pour libérer les ressources.

Paramètres Utilisés

AMPLITUDE_API_URI : URI de base pour l'API Amplitude, utilisé pour configurer le client REST.

Remarques

Gestion des Erreurs : La classe gère diverses exceptions liées aux appels API, en enregistrant les erreurs et en fournissant des messages d'erreur détaillés.

Extensibilité : La classe est conçue pour être extensible, permettant l'ajout de nouvelles fonctionnalités ou l'intégration avec d'autres services API.

Sécurité : Utilise un en-tête x-api-key pour l'authentification lors des appels API.
