PARAMETRAGE DE L’AFFICHAGE DU CHAMP ESCOMPTE

Introduction

Ce paramétrage permet de contrôler l’affichage du champ Escompte dans le formulaire de saisie d’une remise de chèques. Ce champ est utilisé lorsque certaines institutions financières offrent aux clients la possibilité d’escompter un effet ou un chèque, c’est-à-dire d’en recevoir la valeur avant la date d’échéance prévue, sous certaines conditions.

Ce paramètre est particulièrement utile pour activer ou masquer dynamiquement cette fonctionnalité dans l’interface, selon les produits ou services offerts par l’établissement, sans nécessiter de modification du code source.

Affichage du champ Escompte

Nom du paramètre : SHOW_ESCOMPTE

Valeur par défaut : OUI

Libellé : Affiche le champ Escompte sur la remise

Type : CODE_PARAMS

Description :
Ce paramètre contrôle la visibilité du champ Escompte lors de la saisie d’une remise de chèques dans l’application.

  • Si la valeur est OUI, le champ Escompte est affiché à l’utilisateur, lui permettant d’indiquer qu’il souhaite escompter le chèque ou l’effet.

  • Si la valeur est NON, le champ est masqué et l’option d’escompte n’est pas proposée dans l’interface.

Ce mécanisme apporte une souplesse dans la gestion des interfaces utilisateur en s’adaptant aux services réellement utilisés ou proposés par l’institution bancaire.
