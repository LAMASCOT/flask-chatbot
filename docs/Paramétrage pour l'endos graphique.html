PARAMETRAGE POUR L’ENDOS GRAPHIQUE

Introduction

Ce paramétrage permet de configurer les informations nécessaires à l’application du cachet graphique (image d’endossement) sur les chèques. Ces paramètres sont utilisés par les composants chargés du traitement ou de l’impression des documents bancaires. Ils assurent que l’image du cachet est positionnée avec précision à l’endroit souhaité sur le chèque, conformément aux exigences de présentation de l’agence.

Pour fonctionner correctement, le moteur graphique utilisé par l’application doit disposer du chemin vers l’image à insérer, ainsi que des coordonnées X et Y pour positionner cette image sur le document.

Endos graphique corporate

Introduction

L’application offre la possibilité d’apposer un cachet graphique (image d’endossement) sur les chèques lors de leur traitement. Cette fonctionnalité repose sur trois paramètres fondamentaux définis sous le type CODE_PARAMS. Ils permettent de configurer le chemin de l’image utilisée ainsi que son positionnement précis sur le chèque.

Ces réglages garantissent une présentation homogène et conforme aux exigences visuelles de l’établissement bancaire.

Chemin de l’image du cachet

Nom du paramètre : AGENCE_TATTOO_PIC
Valeur par défaut : C:/smartaccess/cachets/EndoCheque.jpg
Libellé : Chemin où se trouve l’endos chèque
Type : CODE_PARAMS

Description :
Ce paramètre indique le chemin absolu vers l’image à utiliser comme cachet sur le chèque. L’application accède à cette ressource pour la superposer au document lors de l’impression ou du traitement numérique.
Le chemin doit être valide et accessible par le système d’exploitation hôte pour garantir un fonctionnement sans erreur.

Position verticale de l’image

Nom du paramètre : TATTOOIMG_Y
Valeur par défaut : 100
Libellé : Position verticale du cachet
Type : CODE_PARAMS

Description :
Spécifie la coordonnée Y (verticale) à laquelle l’image du cachet sera placée sur le chèque. Cette position permet d’ajuster le cachet dans une zone précise, tout en respectant les normes de marquage.
L’unité est exprimée en pixels ou en points selon le moteur graphique utilisé.

Position horizontale de l’image

Nom du paramètre : TATTOOIMG_X
Valeur par défaut : 100
Libellé : Position horizontale du cachet
Type : CODE_PARAMS

Description :
Détermine la coordonnée X (horizontale) de l’image du cachet. Ce paramètre permet d’ancrer l’image à un emplacement spécifique sur le chèque pour un rendu précis et esthétique.
Une mauvaise configuration de cette valeur peut entraîner un chevauchement ou une mauvaise lisibilité du cachet.

Conclusion

Ces trois paramètres sont essentiels pour assurer l’application automatique, conforme et normalisée du cachet graphique sur les chèques.
Toute modification de ces valeurs doit faire l’objet de tests rigoureux afin de garantir que la lisibilité du document et la conformité aux standards graphiques de l’établissement sont préservées.

Endos graphique “compensé le”

Introduction

L’endos graphique “Compensé le” repose sur un ensemble de paramètres essentiels définis dans le système sous le type CODE_PARAMS. Ces paramètres permettent à l’application de positionner et d’appliquer automatiquement un cachet (image ou texte) sur les chèques, avec une grande précision.
Ce mécanisme garantit l’uniformité visuelle et la conformité des chèques endossés selon les exigences des institutions bancaires.

Paramètres de configuration

Nom du paramètre : TATOO_PICTURE
Valeur par défaut : true
Libellé : Flocage du texte
Type : CODE_PARAMS

Description :
Ce paramètre active l’incrustation (flocage) automatique du texte “Compensé le” sur le chèque. Si la valeur est true, le cachet est appliqué automatiquement.

Nom du paramètre : TATOO_PRE_TEXT
Valeur par défaut : ECOBANK SN, COMPENSÉ LE
Libellé : Texte affiché sur l’endos du chèque
Type : CODE_PARAMS

Description :
Définit le contenu du texte qui sera affiché sur le chèque. Il est généralement composé du nom de la banque suivi de la mention “COMPENSÉ LE”.

Nom du paramètre : TATOO_STYLE
Valeur par défaut : Arial-BOLD-12
Libellé : Style de la police du texte
Type : CODE_PARAMS

Description :
Spécifie le style typographique utilisé pour le texte endossé. Cela inclut la police, le poids (gras) et la taille.

Nom du paramètre : TATOO_FOR_SEE
Valeur par défaut : OUI
Libellé : Visualisation
Type : CODE_PARAMS

Description :
Permet d’activer l’option de visualisation du cachet “compensé le” avant impression. Si la valeur est OUI, un aperçu est disponible.

Nom du paramètre : TATOO_X
Valeur par défaut : 100
Libellé : Position horizontale du cachet
Type : CODE_PARAMS

Description :
Définit la position horizontale (coordonnée X) à laquelle le cachet sera appliqué sur le chèque. L’unité est exprimée en pixels ou en points selon le moteur d’impression.

Nom du paramètre : TATOO_Y
Valeur par défaut : 250
Libellé : Position verticale du cachet
Type : CODE_PARAMS

Description :
Spécifie la position verticale (coordonnée Y) de l’endos sur le chèque, permettant un positionnement exact du texte ou de l’image.

Conclusion

Ces paramètres agissent ensemble pour assurer que le cachet “compensé le” est appliqué à l’emplacement exact prévu sur chaque chèque, selon un format cohérent et conforme aux standards de l’agence. Ce système permet également un ajustement rapide sans modification du code source, renforçant la souplesse opérationnelle.
