Introduction

Description : La tache ImalChequeObject est un modèle de données utilisé pour représenter les informations d'un chèque dans le contexte du système IMAL. Elle est conçue pour être sérialisée en JSON, facilitant ainsi l'échange de données entre les systèmes.

Package

Emplacement : sas.action.writers.forimal

Description Fonctionnelle

Objectif : Représenter les informations d'un chèque de manière structurée pour l'intégration avec le système IMAL.

Traitement :

Fournit des getters et setters pour accéder et modifier les attributs du chèque.

Utilise l'annotation @JsonInclude pour exclure les champs null lors de la sérialisation JSON.

Attributs

accountNumberFrom : Numéro de compte de l'émetteur du chèque.

accountNumberTo : Numéro de compte du bénéficiaire du chèque.

amount : Montant du chèque.

benef_name : Nom du bénéficiaire.

chq_number : Numéro du chèque.

chq_date : Date du chèque.

recto : Image ou information du recto du chèque.

verso : Image ou information du verso du chèque.

trs_no : Numéro de transaction associé au chèque.

micr : Code MICR du chèque.

Méthodes

ImalChequeObject() : Constructeur par défaut.

Getters et Setters : Pour chaque attribut, permettant l'accès et la modification des valeurs.

toString() : Retourne une représentation en chaîne de caractères de l'objet, incluant tous ses attributs.

Paramètres Utilisés

Aucun paramètre spécifique mentionné : La tache est un modèle de données et ne dépend pas de paramètres externes pour son fonctionnement. Elle est conçue pour être utilisée dans des contextes où les données de chèque doivent être échangées avec le système IMAL ou d'autres systèmes compatibles JSON.

Remarques

Sérialisation JSON : La tache est configurée pour exclure les champs null lors de la conversion en JSON, ce qui est utile pour réduire la taille des données échangées et éviter les valeurs indésirables.

Extensibilité : La tache peut être étendue pour inclure des attributs supplémentaires si nécessaire pour d'autres intégrations ou fonctionnalités.

Intégration : Conçue pour être utilisée dans des contextes où les données de chèque doivent être échangées avec le système IMAL ou d'autres systèmes compatibles JSON.
