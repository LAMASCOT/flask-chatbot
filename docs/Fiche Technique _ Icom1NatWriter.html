Introduction

Description : La tache Icom1NatWriter est une extension de FlatFileWriter utilisée pour générer et envoyer des fichiers ICOM1 nationaux vers la BCEAO. Elle traite divers types d'opérations financières telles que les chèques, virements, effets, et prélèvements, en fonction de leur état et type.

Package

Emplacement : sas.action.writers.sica

Description Fonctionnelle

Objectif : Générer et envoyer des fichiers ICOM1 nationaux vers la BCEAO en traitant les opérations financières disponibles.

Traitement :

Vérifie les paramètres de compensation manuelle et d'envoi.

Récupère les opérations financières (chèques, virements, effets, prélèvements) d'une banque.

Prépare les opérations en fonction de leur type et destination (Banque à Banque, Banque à SFD, SFD à Banque, SFD à SFD).

Met à jour l'état des opérations après traitement.

Méthodes

Icom1NatWriter() : Constructeur par défaut qui initialise la description de l'envoi.

execute() : Exécute le processus de génération et d'envoi des fichiers ICOM1.

feedOperationWithBltOrdre(int j, Operation[] operations, Effets effet) : Prépare une opération avec un billet à ordre.

feedOperationWithBltOrdre45(int j, Operation[] operations, Effets effet) : Prépare une opération avec un billet à ordre normalisé.

feedOperationWithCheque(int j, Cheques cheque, Operation[] operations) : Prépare une opération avec un chèque.

feedOperationWithCheque35(int j, Cheques cheque, Operation[] operations) : Prépare une opération avec un chèque de type 35.

feedOperationWithLtrChange(Operation[] operations, int j, Effets effet) : Prépare une opération avec une lettre de change.

feedOperationWithLtrChange46(Operation[] operations, int j, Effets effet) : Prépare une opération avec une lettre de change normalisée.

feedOperationWithPrelevement(int j, Operation[] operations, Prelevements prelevement) : Prépare une opération avec un prélèvement.

feedOperationWithVirement10(Virements virement, int j, Operation[] operations) : Prépare une opération avec un virement de type 10.

Paramètres Utilisés

CETAOPEALLICOM1 : État des opérations à traiter.

CHEACCIMASCANOUNOR : Type de chèque à traiter.

VIRSTACLIANCNOR : Type de virement clientèle à traiter.

VIRSTACLINOUNOR : Type de virement non normalisé à traiter.

VIRBANBANANCNOR : Type de virement banque à banque à traiter.

VIRMISDISANCNOR : Type de virement mise à disposition à traiter.

VIRMISDISNOUNOR : Type de virement mise à disposition non normalisé à traiter.

BILORDNOUNOR : Type d'effet billet à ordre non normalisé à traiter.

LETCHANOUNOR : Type d'effet lettre de change non normalisé à traiter.

ORDPRENOUNOR : Type de prélèvement non normalisé à traiter.

EFFET_NORMALISE : Indicateur d'effet normalisé.

COMPENSATION_MANUELLE : Indicateur de compensation manuelle.

Remarques

Gestion des Erreurs : Les erreurs lors de la génération ou de l'envoi des fichiers sont gérées par des logs et des mises à jour de paramètres.

Extensibilité : La tache peut être étendue pour inclure d'autres types d'opérations ou de traitements spécifiques.

Intégration : Cette tache utilise DataBaseJPA pour interagir avec la base de données et récupérer les opérations à traiter.
