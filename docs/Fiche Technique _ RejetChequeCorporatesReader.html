Introduction

Description : La tache RejetChequeCorporatesReader est conçue pour lire et traiter les fichiers plats contenant des informations sur les chèques rejetés, en mettant à jour les enregistrements dans la base de données.

Package

Emplacement : sas.action.readers.corporates

Description Fonctionnelle

Déclenchement : Activée lors de la détection de nouveaux fichiers plats dans le répertoire configuré.

Traitement :

Lit le fichier pour extraire les informations de rejet de chèques.

Met à jour les enregistrements de chèques dans la base de données en fonction des informations de rejet.

Met à jour les remises associées en fonction des chèques rejetés.

Résultat : Les chèques rejetés sont traités et mis à jour dans la base de données.

En cas d’erreur : Les erreurs de traitement sont consignées dans les journaux pour un suivi ultérieur.

Opérations Clés

Lecture des Fichiers : Utilise BufferedReader pour lire les lignes du fichier et extraire les données.

Mise à Jour des Enregistrements :

Utilise DataBaseJPA pour mettre à jour les enregistrements de chèques et de remises dans la base de données.

Met à jour l'état et le motif de rejet des chèques en fonction des informations extraites.

Insère les chèques rejetés dans une table dédiée et les supprime de la table principale.

Met à jour les remises associées en ajustant le nombre d'opérations et le montant total.

Paramètres du Composant

Répertoires : Utilise des objets Repertoires pour gérer les répertoires de fichiers.

Paramètres de Configuration :

CETAOPEANO pour la gestion des états de rejet des chèques et des remises.

Interactions avec la Base de Données

Requêtes SQL : Utilise des requêtes SQL pour mettre à jour les enregistrements de chèques et de remises.

Gestion des Transactions : Utilise DataBaseJPA pour gérer les transactions de base de données.

Paramètres Utilisés et Valeurs Possibles

CETAOPEANO : État des chèques et remises rejetés.

Remarques

Implémentation Future : La méthode treatFile est partiellement implémentée et peut nécessiter des ajustements pour gérer de nouveaux types de rejets ou de nouvelles exigences de traitement.
