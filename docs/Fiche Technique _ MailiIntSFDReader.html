Introduction

Description : La tache MailiIntSFDReader est conçue pour lire et traiter les fichiers binaires contenant des informations sur les remises de chèques et d'effets pour les clients SFD (Systèmes Financiers Décentralisés), en mettant à jour les enregistrements dans la base de données et en intégrant les valeurs dans le système bancaire.

Package

Emplacement : sas.action.readers.corporates

Description Fonctionnelle

Déclenchement : Activée lors de la détection de nouveaux fichiers binaires dans le répertoire configuré.

Traitement :

Lit le fichier pour extraire les en-têtes et les index d'images.

Met à jour ou insère les enregistrements de chèques et d'effets dans la base de données.

Gère les remises de chèques et d'effets, en vérifiant les doublons et en mettant à jour les états.

Envoie des notifications par email si configuré.

Résultat : Les enregistrements de chèques et d'effets sont mis à jour dans la base de données, et les notifications sont envoyées.

En cas d’erreur : Les erreurs de traitement sont consignées dans les journaux pour un suivi ultérieur.

Opérations Clés

Lecture des Fichiers : Utilise des flux d'entrée pour lire les en-têtes et les index d'images des fichiers binaires.

Mise à Jour des Enregistrements :

Utilise DataBaseJPA pour exécuter des requêtes SQL qui mettent à jour ou insèrent les enregistrements de chèques et d'effets dans la base de données.

Gère les doublons en insérant les enregistrements dans une table de doublons.

Notifications : Envoie des emails récapitulatifs si configuré.

Paramètres du Composant

`CETAOPEVAL2` : Code d'état pour les opérations validées.

`CETAOPEVAL` : Code d'état pour les opérations en cours de validation.

`CETAOPEALLICOM1` : Code d'état pour les opérations en attente de validation.

`CETAREMERR` : Code d'état pour les remises en erreur.

`CETAIMASTO` : Code d'état pour les images stockées.

Interactions avec la Base de Données

Requêtes SQL : Utilise des requêtes SQL pour mettre à jour ou insérer les enregistrements de chèques et d'effets dans la base de données.

Gestion des Transactions : Utilise DataBaseJPA pour gérer les transactions de base de données.

Paramètres Utilisés et Valeurs Possibles

`CORPORATE_OFFLINE` :

Utilisation : Indique si le mode hors ligne est activé pour les opérations corporatives.

Valeurs Possibles : "1" pour activé, autre pour désactivé.

`VALIDATION_CORP_CHEQUE_AUTO` :

Utilisation : Indique si la validation automatique des chèques corporatifs est activée.

Valeurs Possibles : "AUTO" pour activé, autre pour désactivé.

`SEND_RECAP_EMAIL` :

Utilisation : Indique si un email récapitulatif doit être envoyé.

Valeurs Possibles : "OUI" pour activé, autre pour désactivé.
