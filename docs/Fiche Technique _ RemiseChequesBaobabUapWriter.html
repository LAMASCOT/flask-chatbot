Introduction

Description : La tache RemiseChequesBaobabUapWriter est une extension de BrmWriter utilisée pour gérer l'envoi des remises et des chèques vers le système UAP BAOBAB. Elle traite les remises et les chèques, met à jour leur état, et génère les valeurs nécessaires pour l'intégration avec UAP BAOBAB.

Package

Emplacement : sas.action.writers.baobab_uap

Description Fonctionnelle

Objectif : Gérer l'envoi des remises et des chèques vers UAP BAOBAB, en mettant à jour les états et en générant les valeurs nécessaires pour l'intégration.

Traitement :

Récupère les remises et les chèques à traiter.

Met à jour l'état des remises et des chèques après traitement.

Génère les valeurs pour l'intégration avec UAP BAOBAB.

Gère les deux sens d'interface : "ALLER" pour l'envoi des remises et "RETOUR" pour l'envoi des chèques rejetés.

Méthodes

execute() : Exécute le processus d'envoi des remises et des chèques, en récupérant les remises et les chèques à traiter, en mettant à jour leur état, et en générant les valeurs pour UAP BAOBAB.

generateValuesForUapBaobab(Remises remise) : Génère les valeurs pour l'envoi des remises vers UAP BAOBAB.

generateRejectValuesForUapBaobab(List<Cheques> cheques) : Génère les valeurs pour l'envoi des chèques rejetés vers UAP BAOBAB.

convertFromRemises(Remises r) : Convertit une remise en un objet IntRemisesAller pour l'intégration avec UAP BAOBAB.

convertFromOneCheque(Cheques c) : Convertit un chèque en un objet IntChequesAller pour l'intégration avec UAP BAOBAB.

convertFromOneChequeRej(Cheques c) : Convertit un chèque rejeté en un objet IntChequesRejetes pour l'intégration avec UAP BAOBAB.

Paramètres Utilisés

CODE_BANQUE_SICA3 : Code de la banque utilisé pour identifier l'établissement.

CETAOPEVAL2 : Paramètre utilisé pour identifier l'état des remises à traiter.

CETAOPEALLICOM1 : Paramètre utilisé pour mettre à jour l'état des chèques après traitement.

CETAOPEALLICOM2 : Paramètre utilisé pour identifier l'état des chèques rejetés à traiter.

CETAOPEALLICOM2ENV : Paramètre utilisé pour mettre à jour l'état des chèques rejetés après traitement.

LOCK_CHK_ALL_COMPENSE : Paramètre utilisé pour verrouiller le traitement des chèques.

LOCK_CHK_RET_RJT_COMPENSE : Paramètre utilisé pour verrouiller le traitement des chèques rejetés.

Remarques

Gestion des Erreurs : Les erreurs lors du traitement des remises et des chèques sont enregistrées dans les logs et peuvent déclencher des notifications d'erreur.

Extensibilité : La classe est conçue pour être facilement extensible pour inclure d'autres types de vérifications ou de formats de fichiers si nécessaire.

Configuration : La classe utilise des composants configurables pour le traitement des remises et des chèques, permettant une adaptation facile aux besoins spécifiques du système.

Interface Sens : La classe gère deux sens d'interface : "ALLER" pour l'envoi des remises et "RETOUR" pour l'envoi des chèques rejetés.
