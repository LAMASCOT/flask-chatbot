Introduction

Description : La tache ImalServiceUtils fournit des utilitaires pour interagir avec le système IMAL, notamment pour la gestion des chèques. Elle permet de créer, mettre à jour et récupérer le statut des chèques dans IMAL.

Package

Emplacement : sas.action.writers.forimal

Description Fonctionnelle

Objectif : Fournir des méthodes utilitaires pour gérer les interactions avec le système IMAL concernant les chèques.

Traitement :

Récupère la date d'endossement via un service IMAL.

Crée un chèque dans le système IMAL.

Met à jour un chèque avec des images dans IMAL.

Récupère le statut d'un chèque dans IMAL.

Méthodes

ImalServiceUtils() : Constructeur par défaut.

returnDateEndos() : Récupère la date d'endossement à partir du service IMAL.

createChequeInImal(Cheques cheque) : Crée un chèque dans le système IMAL et met à jour ses informations.

updateChequeInImal(Cheques cheque) : Met à jour un chèque avec des images dans le système IMAL.

fetchChequeStatusInImal(Cheques cheque) : Récupère le statut d'un chèque dans le système IMAL et met à jour ses informations en conséquence.

Attributs

LOG : Logger pour enregistrer les événements et les erreurs.

dataBaseJPA : Instance de DataBaseJPA pour interagir avec la base de données.

Paramètres Utilisés

CETAOPEALLICOM1 : État des chèques après mise à jour avec image.

CETAOPEALLICOM1ACC : État des chèques validés par BIS.

CETAOPEALLICOM1ACCENVSIB : État des chèques après envoi vers SIB.

CETAOPEANO : État des chèques annulés ou rejetés.

CETAOPEVAL2 : État des chèques en cours de validation.

CETAOPEREJRET : État des chèques rejetés après compensation.

DATECOMPENS_NAT : Date de compensation nationale.

DATECOMPENS_SRG : Date de compensation régionale.

CODE_SICA3 : Code SICA3 pour la génération du MICR.

Remarques

Gestion des Erreurs : Les erreurs lors des interactions avec le système IMAL sont gérées et enregistrées dans les logs.

Extensibilité : La tache peut être étendue pour inclure d'autres interactions ou services liés à IMAL.

Intégration : Conçue pour être utilisée dans des contextes où les chèques doivent être gérés et suivis dans le système IMAL.
