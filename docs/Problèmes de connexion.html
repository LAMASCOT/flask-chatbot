Problème

Décrivez brièvement le problème (ex. : "Erreur 500 lors de l’accès à l'application").

Symptômes

  • L'application affiche une page blanche.

  • Un message d’erreur HTTP apparaît.

Causes possibles

  1. Problème de base de données.

  2. Service backend non démarré.

Procédure de diagnostic

  1. Vérifier les logs d'application : tail -f /var/log/app.log.

  2. Vérifier l'état du service avec systemctl status app.service.

Solution rapide

  • Redémarrer le service avec la commande : systemctl restart app.service.

  • Vérifier la connexion à la base de données avec ping db-server.

Escalade

Si le problème persiste, contacter l’équipe développement via devs@smartaps.tech
