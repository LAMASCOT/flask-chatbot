Introduction

Description : La tache LotFinacleUBACIReader est conçue pour lire et traiter les fichiers de lots provenant du système Finacle, en extrayant les informations nécessaires et en mettant à jour les enregistrements dans la base de données.

Package

Emplacement : sas.action.readers.delta

Description Fonctionnelle

Déclenchement : Activée lors de la détection de nouveaux fichiers de lots dans le répertoire configuré.

Traitement :

Lit chaque ligne du fichier pour extraire les informations des opérations bancaires.

Crée des objets EnteteLot et Operation à partir des lignes lues.

Vérifie si le fichier a déjà été traité en consultant la base de données.

Met à jour les enregistrements dans la base de données si les conditions sont remplies.

Résultat : Les enregistrements sont mis à jour dans la base de données.

En cas d’erreur : Les erreurs de traitement sont consignées dans les journaux pour un suivi ultérieur.

Opérations Clés

Lecture des Fichiers : Utilise un BufferedReader pour lire chaque ligne du fichier et extraire les informations nécessaires.

Création d'Objets :

Utilise la méthode getChamp pour extraire les champs des lignes et créer des objets EnteteLot et Operation.

Vérification de Traitement :

Vérifie si le fichier a déjà été traité en consultant la base de données.

Mise à Jour des Enregistrements :

Met à jour les enregistrements dans la base de données si les conditions sont remplies.

Paramètres du Composant

Séparateur de Ligne : Utilise des méthodes spécifiques pour diviser les lignes en attributs.

Interactions avec la Base de Données

Requêtes SQL : Utilise des requêtes SQL pour vérifier si le fichier a déjà été traité et pour insérer de nouveaux enregistrements.

Gestion des Transactions : Utilise DataBase pour gérer les transactions de base de données.

Paramètres Utilisés et Valeurs Possibles

`CETAFICTRASUC` :

Utilisation : Code d'état pour les fichiers traités avec succès.

Valeurs Possibles : Identifiants numériques définis dans le système.

`IDFICHIER` :

Utilisation : Identifiant unique pour chaque fichier traité.

Valeurs Possibles : Généré dynamiquement pour chaque fichier.

`patternDate` :

Utilisation : Format de date utilisé pour le traitement des dates.

Valeurs Possibles : Chaînes de format de date, par exemple, "yyyy/MM/dd".
