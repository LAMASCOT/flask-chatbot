Introduction

Description : La tache PrelevementUploadReader est conçue pour lire et traiter les fichiers plats contenant des informations de prélèvements à uploader, en vérifiant et en préparant les données pour l'insertion dans la base de données.

Package

Emplacement : sas.action.readers.corporates

Description Fonctionnelle

Déclenchement : Activée lors de la détection de nouveaux fichiers plats dans le répertoire configuré.

Traitement :

Lit le fichier pour extraire les informations de prélèvement.

Vérifie la validité des données, notamment la clé RIB.

Prépare les données pour l'insertion dans la base de données.

Résultat : Les prélèvements sont vérifiés et prêts pour l'insertion dans la base de données.

En cas d’erreur : Les erreurs de traitement, telles que les erreurs de clé RIB ou les montants incorrects, sont consignées dans les journaux pour un suivi ultérieur.

Opérations Clés

Lecture des Fichiers : Utilise BufferedReader pour lire les lignes du fichier et extraire les données.

Vérification des Données :

Vérifie la clé RIB calculée par rapport à celle fournie dans le fichier.

Compare le montant total calculé avec celui indiqué dans le fichier.

Préparation des Données :

Prépare les objets Prelevements avec les informations extraites pour une insertion ultérieure dans la base de données.

Paramètres du Composant

Répertoires : Utilise des objets Repertoires pour gérer les répertoires de fichiers.

Paramètres de Configuration :

ORDPRENOUNOR, CODE_BANQUE_SICA3, CETAOPEVALDELTA pour la gestion des prélèvements.

Interactions avec la Base de Données

Préparation des Données : Les objets Prelevements sont préparés avec les informations nécessaires pour une insertion ultérieure dans la base de données.

Paramètres Utilisés et Valeurs Possibles

ORDPRENOUNOR : Type de prélèvement.

CODE_BANQUE_SICA3 : Code de la banque utilisé pour les prélèvements.

CETAOPEVALDELTA : État initial des prélèvements à uploader.

Remarques

Implémentation Future : La méthode treatFile est partiellement implémentée et peut nécessiter des ajustements pour gérer de nouveaux types de prélèvements ou de nouvelles exigences de traitement.
