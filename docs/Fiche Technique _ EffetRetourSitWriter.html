Introduction

Description : La tache EffetRetourSitWriter est une extension de FlatFileWriter utilisée pour envoyer les effets retour vers le SIB (Système d'Information Bancaire). Elle génère un fichier plat contenant les informations des effets retour et met à jour leur état dans la base de données.

Package

Emplacement : sas.action.writers.orion

Description Fonctionnelle

Objectif : Envoyer les effets retour vers le SIB en générant un fichier plat et mettre à jour leur état dans la base de données.

Traitement :

Vérifie si le verrouillage des effets retour est activé.

Récupère les effets retour à envoyer.

Génère un fichier plat avec les informations des effets retour.

Met à jour l'état des effets après envoi.

Gère le verrouillage des effets retour pour éviter les conflits d'accès.

Méthodes

EffetRetourSitWriter() : Constructeur par défaut qui initialise la description de l'envoi.

execute() : Exécute le processus d'envoi des effets retour vers le SIB.

getMessageEffetsRecus() : Retourne le message des effets reçus.

setMessageEffetsRecus(String messageEffetsRecus) : Définit le message des effets reçus.

Paramètres Utilisés

LOCK_EFF_RETOUR : Paramètre de verrouillage pour éviter les conflits d'accès lors de la composition des fichiers d'effets retour.

CETAOPERETREC : État des effets retour reçus.

CETAOPERET : État des effets retour.

CETAOPERETENVSIB : État des effets après envoi vers le SIB.

CETAOPERETRECENVSIB : État des effets retour reçus après envoi vers le SIB.

SIB_IN_FOLDER : Dossier de destination pour les fichiers générés.

EFF_OUT_FILE_ROOTNAME : Nom de base pour les fichiers d'effets retour.

Remarques

Gestion des Erreurs : Les erreurs lors de l'envoi des effets retour sont gérées par des logs et des messages d'avertissement.

Extensibilité : La tache peut être étendue pour inclure d'autres types d'effets ou de traitements spécifiques.

Intégration : Cette tache utilise DataBaseJPA pour interagir avec la base de données et effectuer les mises à jour nécessaires.
