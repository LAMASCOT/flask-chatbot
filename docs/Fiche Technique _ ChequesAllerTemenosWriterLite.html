Introduction

Description : La tache ChequesAllerTemenosWriterLite est une extension de FlatFileWriter utilisée pour générer et envoyer des fichiers de chèques vers le système SIB (Système d'Information Bancaire) dans le cadre d'une migration vers Temenos. Elle traite les remises de chèques et génère un fichier plat contenant les informations nécessaires pour le traitement. Cette version "Lite" semble être optimisée pour un traitement plus léger ou spécifique.

Package

Emplacement : sas.action.writers.temenos

Description Fonctionnelle

Objectif : Générer un fichier contenant les informations des chèques à envoyer vers le SIB pour une migration vers Temenos.

Traitement :

Récupère les remises de chèques depuis la base de données.

Calcule le nombre de chèques.

Génère un fichier plat avec les informations des remises et des chèques.

Met à jour l'état des remises et des chèques dans la base de données après traitement.

Attributs

remise : Remises - Instance de la remise actuellement traitée.

db : DataBase - Instance de la base de données utilisée pour récupérer et mettre à jour les informations des remises et des chèques.

Méthodes

ChequesAllerTemenosWriterLite() : Constructeur par défaut qui initialise la description de l'opération.

ChequesAllerTemenosWriterLite(Remises remise) : Constructeur qui initialise la classe avec une remise spécifique.

execute() : Exécute le processus de génération et d'envoi des fichiers de chèques vers le SIB.

Paramètres Utilisés

CETAOPEERR : État des opérations, utilisé pour marquer les chèques avec des erreurs.

CETAOPEVAL : État des opérations, utilisé pour filtrer les chèques à traiter.

SIB_IN_FOLDER : Répertoire de destination pour les fichiers générés, où les fichiers de chèques seront stockés après leur création.

CHQ_IN_FILE_ROOTNAME : Nom de base pour les fichiers de chèques générés.

SIB_FILE_EXTENSION : Extension des fichiers de chèques générés.

Remarques

Gestion des Erreurs : Les erreurs lors de la génération des fichiers sont gérées par des logs, et la description de l'opération est mise à jour pour refléter le statut de chaque étape.

Extensibilité : La classe peut être étendue pour inclure d'autres types de remises ou pour intégrer d'autres systèmes de traitement de chèques.

Utilisation de la Base de Données : La classe interagit directement avec la base de données pour récupérer et mettre à jour les informations des remises et des chèques, en utilisant des requêtes SQL pour la persistance des données.
