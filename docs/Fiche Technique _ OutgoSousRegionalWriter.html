Introduction

Description : La tache OutgoSousRegionalWriter est une extension de FlatFileWriter utilisée pour générer et envoyer des fichiers de transactions sous-régionales vers la BCEAO. Elle traite différents types de transactions, y compris les chèques, virements, effets, et prélèvements, en les organisant en lots pour l'envoi.

Package

Emplacement : sas.action.writers.forimal

Description Fonctionnelle

Objectif : Générer des fichiers de transactions sous-régionales pour la BCEAO en traitant divers types de transactions bancaires.

Traitement :

Récupère les transactions à partir de la base de données en fonction de leur état et type.

Organise les transactions en lots et les prépare pour l'envoi.

Gère les rejets et les erreurs de traitement.

Méthodes

OutgoSousRegionalWriter() : Constructeur par défaut.

execute() : Exécute le processus de génération et d'envoi des fichiers de transactions sous-régionales.

prepareCheques() : Prépare les chèques pour l'envoi en les organisant en lots.

prepareVirements() : Prépare les virements pour l'envoi en les organisant en lots.

prepareEffets() : Prépare les effets pour l'envoi en les organisant en lots.

preparePrelevements() : Prépare les prélèvements pour l'envoi en les organisant en lots.

prepareRejetEffets() : Prépare les effets rejetés pour l'envoi en les organisant en lots.

printRemise() : Imprime les remises préparées pour l'envoi.

Attributs

sql : Requête SQL pour récupérer les transactions.

refLot : Référence du lot courant.

vEnteteLots : Liste des entêtes de lots préparés pour l'envoi.

sequence : Séquence utilisée pour l'identification des lots.

enteteRemise : Entête de la remise courante.

cheques, virements, effets, prelevements : Listes des transactions récupérées pour le traitement.

remcom, lotcom : Objets représentant les remises et lots courants.

remiseHasCheques, remiseHasEffets, remiseHasVirements, remiseHasPrelevements, remiseHasRejetEffets : Indicateurs de la présence de différents types de transactions dans la remise.

db : Instance de DataBaseJPA pour interagir avec la base de données.

Paramètres Utilisés

CETAOPERET, CETAOPERETREC, CETAOPEREJRET : États des opérations pour filtrer les transactions.

CHEACCIMASCANOUNOR, VIRSTACLIANCNOR, VIRSTACLINOUNOR, VIRBANBANANCNOR, VIRMISDISANCNOR, VIRMISDISNOUNOR, BILORDNOUNOR, LETCHANOUNOR, ORDPRENOUNOR : Types de transactions pour filtrer les transactions.

patternDate : Format de date utilisé pour les conversions de date.

Remarques

Gestion des Erreurs : Les erreurs lors du traitement des transactions sont gérées et enregistrées dans les logs.

Extensibilité : La tache peut être étendue pour inclure d'autres types de transactions ou traitements spécifiques.

Intégration : Conçue pour être utilisée dans des contextes où les transactions sous-régionales doivent être gérées et suivies dans le système bancaire, en respectant les règles définies par la BCEAO.
