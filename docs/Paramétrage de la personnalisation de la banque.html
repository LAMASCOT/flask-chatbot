PARAMETRAGE DE LA PERSONNALISATION DE LA BANQUE

Introduction

Ce paramétrage permet de personnaliser l’identité visuelle et les messages d’accueil de l’application bancaire. Il est utilisé pour adapter l’expérience utilisateur aux spécificités de la banque déployant la solution SmartAccess. Ces paramètres influencent directement l’apparence de l’interface utilisateur ainsi que les premiers éléments de communication affichés à l’ouverture de l’application.

Ils jouent un rôle essentiel dans la cohérence de l’image de marque et dans la clarté des échanges avec l’utilisateur final.

Nom de la banque

APP_NAME_IN_BANK

  • Valeur : smartaccess

  • Libellé par défaut: BGFICLEARING

  • Type : CODE_PARAMS

  • Description : Ce paramètre permet de définir le nom de l'application bancaire tel qu’il doit apparaître dans l’interface. Par exemple, bien que l’application soit générique (smartaccess), elle peut être personnalisée pour afficher le nom spécifique de la banque, ici BGFICLEARING. Cela renforce la personnalisation de l’outil pour chaque client institutionnel.

Message de bienvenue

WELCOME_MESSAGE

  • Valeur par défaut : Connectez-vous à

  • Libellé : Message de bienvenue

  • Type : CODE_PARAMS

  • Description : Ce paramètre définit le texte affiché lors de la connexion initiale à l'application. Il s’agit d’un message d’accueil ou d’invitation à se connecter. Ce message peut être personnalisé pour refléter le ton et le style de communication de la banque.

Ces paramètres permettent d’adapter dynamiquement l’identité visuelle et textuelle de l’application selon la banque cliente, sans nécessiter de modification dans le code source. Cela favorise une réutilisabilité optimale de la plateforme pour plusieurs entités bancaires.
