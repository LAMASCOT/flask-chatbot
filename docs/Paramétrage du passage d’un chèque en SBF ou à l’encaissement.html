PARAMETRAGE DU PASSAGE D’UN CHEQUE EN SBF OU A L’ENCAISSEMENT

Introduction

Ce paramétrage permet de contrôler le comportement de l’application lors de la validation d’une remise de chèques, en définissant si les chèques doivent être automatiquement passés à l’encaissement ou en SBF (Sauf bonne fin).
Il s’agit d’un mécanisme flexible permettant aux établissements financiers de s’adapter à leurs procédures internes sans nécessité de modifier le code source. Le paramètre associé permet de choisir dynamiquement le mode de traitement souhaité pour les chèques déposés par les clients.

Mode de traitement des chèques

Nom du paramètre : SHOW_SBF_AE

Valeur par défaut : OUI

Libellé : Affiche les méthodes passer un cheque en SBF , a  l'encaissement dans le menu de Validation des Remises

Type : CODE_PARAMS

Description :
Ce paramètre détermine la méthode de traitement appliquée aux chèques au moment de leur validation dans le menu Validation des Remises.

  • Si la valeur est ENCAISSEMENT, les chèques sont directement traités comme encaissés. Cela signifie qu’ils suivent le circuit standard sans retenue particulière, conformément aux délais usuels d’encaissement.

  • Si la valeur est SBF, les chèques sont enregistrés sous forme de créances conditionnelles. Le montant est crédité de manière prévisionnelle, sous réserve de bonne fin, avec des règles de disponibilités spécifiques.

Ce mécanisme permet de refléter fidèlement les politiques de gestion de risque et les processus internes de traitement des chèques propres à chaque institution. Le changement de mode peut être opéré simplement en modifiant la valeur du paramètre dans le système.
