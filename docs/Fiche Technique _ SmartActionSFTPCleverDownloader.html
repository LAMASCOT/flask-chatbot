Description : La tache SmartActionSFTPCleverDownloader est conçue pour télécharger des fichiers depuis un serveur SFTP suite à une action comme un bouton. Elle gère la connexion sécurisée, le téléchargement des fichiers, et la gestion des erreurs.

Description Fonctionnelle

Déclenchement : La méthode execute() est appelée pour initier le processus de téléchargement.

Traitement : Établit une connexion SFTP sécurisée, télécharge les fichiers spécifiés, et gère les erreurs potentielles.

Résultat : Les fichiers sont téléchargés avec succès depuis le serveur SFTP.

En cas d’erreur : Les erreurs de connexion ou de téléchargement sont consignées dans les journaux.

Opérations Clés

Connexion SFTP : Établit une connexion sécurisée au serveur SFTP en utilisant un nom d'utilisateur et un mot de passe, ou une clé privée.

Téléchargement de Fichiers : Télécharge les fichiers spécifiés depuis le répertoire distant sur le serveur SFTP vers un répertoire local.

Gestion des Erreurs : Gère les erreurs de connexion et de téléchargement.

Suppression des Fichiers : Supprime les fichiers téléchargés du serveur SFTP après un téléchargement réussi.

Paramètres du Composant

SFTPHOST : Adresse du serveur SFTP, obtenue dynamiquement à partir de l'identifiant du répertoire.

SFTPPORT : Port du serveur SFTP, avec une valeur par défaut de 22 si non spécifiée.

SFTPUSER : Nom d'utilisateur pour la connexion SFTP, obtenu dynamiquement.

SFTPPASSWORD : Mot de passe pour la connexion SFTP, obtenu dynamiquement.

REMOTE_FOLDER : Répertoire distant où les fichiers doivent être téléchargés, défini par repertoire.getChemin(), avec comme valeur par défaut /remote/incoming/ (configurable).

LOCAL_FOLDER : Répertoire local où les fichiers téléchargés sont stockés, défini par repertoire.getPartenaire(), avec comme valeur par défaut /var/data/sftp_download.

REMOTE_FILE_EXTENSION : Extension des fichiers à considérer pour le téléchargement, définie par repertoire.getExtension(), avec comme valeur par défaut *.

SFTP_CONN_VIA_KEY : Indique si la connexion SFTP doit se faire via une clé privée, avec comme valeur par défaut false.

SFTP_TIME_ALIVE : Temps d'attente pour la connexion SFTP, avec une valeur par défaut de 300000 ms.

PREFERRED_AUTH_SFTP : Méthode d'authentification préférée pour SFTP, par défaut password.

SFTP_OLD_ALGOS : Indique si des algorithmes plus anciens doivent être utilisés pour la connexion, avec comme valeur par défaut false.

Exemples d'Utilisation

Utilisée pour automatiser le téléchargement sécurisé de fichiers depuis des serveurs SFTP dans un environnement d'entreprise.

Intégration avec d'autres Systèmes

Intégrée avec les systèmes de gestion de fichiers pour assurer un téléchargement sécurisé et fiable des fichiers.
