Introduction

Description : La tache EffetAllerRejeteSitWriter est une extension de FlatFileWriter utilisée pour envoyer les effets rejetés vers le SIB (Système d'Information Bancaire). Elle génère un fichier plat contenant les informations des effets rejetés et met à jour leur état dans la base de données.

Package

Emplacement : sas.action.writers.orion

Description Fonctionnelle

Objectif : Envoyer les effets rejetés vers le SIB en générant un fichier plat et mettre à jour leur état dans la base de données.

Traitement :

Vérifie si le verrouillage des effets rejetés est activé.

Récupère les effets rejetés à envoyer.

Génère un fichier plat avec les informations des effets rejetés.

Met à jour l'état des effets après envoi.

Gère le verrouillage des effets rejetés pour éviter les conflits d'accès.

Méthodes

EffetAllerRejeteSitWriter() : Constructeur par défaut qui initialise la description de l'envoi.

execute() : Exécute le processus d'envoi des effets rejetés vers le SIB.

Paramètres Utilisés

LOCK_EFF_RJT_ALLER : Paramètre de verrouillage pour éviter les conflits d'accès lors de la composition des fichiers d'effets rejetés.

CETAOPEREJRET : État des effets rejetés.

CETAOPEREJRETENVSIB : État des effets après envoi vers le SIB.

SIB_IN_FOLDER : Dossier de destination pour les fichiers générés.

EFF_REJ_FILE_ROOTNAME : Nom de base pour les fichiers d'effets rejetés.

Remarques

Gestion des Erreurs : Les erreurs lors de l'envoi des effets rejetés sont gérées par des logs et des messages d'avertissement.

Extensibilité : La tache peut être étendue pour inclure d'autres types d'effets ou de traitements spécifiques.

Intégration : Cette tache utilise DataBaseJPA pour interagir avec la base de données et effectuer les mises à jour nécessaires.
