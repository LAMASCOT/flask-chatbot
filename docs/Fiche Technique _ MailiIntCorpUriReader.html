Introduction

Description : La tache MailiIntCorpUriReader est conçue pour lire et traiter les fichiers binaires contenant des informations sur les remises de chèques et d'effets, en intégrant ces données dans la base de données et en gérant les interactions avec les systèmes externes.

Package

Emplacement : sas.action.readers.corporates

Description Fonctionnelle

Déclenchement : Activée lors de la détection de nouveaux fichiers de remises dans le répertoire configuré.

Traitement :

Lit les en-têtes et les index d'images des fichiers binaires.

Insère les informations des remises dans la base de données.

Met à jour les états des remises et des chèques/effets en fonction des paramètres configurés.

Gère les interactions avec les systèmes externes tels que REIS et IMAL.

Envoie des emails récapitulatifs si configuré.

Résultat : Les remises et les chèques/effets sont intégrés et mis à jour dans la base de données, et les notifications sont envoyées si nécessaire.

En cas d’erreur : Les erreurs de traitement sont consignées dans les journaux pour un suivi ultérieur.

Opérations Clés

Lecture des Fichiers : Utilise BufferedInputStream pour lire les fichiers binaires.

Mise à Jour des Enregistrements :

Utilise des requêtes SQL pour insérer et mettre à jour les remises et les chèques/effets dans la base de données.

Utilise DataBaseJPA pour exécuter les transactions de base de données.

Interactions Externes :

Intègre les remises avec le système REIS pour le screening.

Crée des chèques dans le système IMAL si activé.

Paramètres du Composant

`IMG_ALLER_FOLDER` : Dossier de destination pour les images traitées.

`EMAILS_DEST_OPS` : Destinataires des emails de notification.

`REIS_ENABLE` : Active ou désactive l'intégration avec le système REIS.

`IMAL_WEBSERVICE_ENABLE` : Active ou désactive l'intégration avec le système IMAL.

`SPLIT_REMISE` : Indique si les remises doivent être divisées en fonction des types de chèques.

Interactions avec la Base de Données

Requêtes SQL : Utilise des requêtes SQL pour insérer et mettre à jour les remises et les chèques/effets.

Gestion des Transactions : Utilise DataBaseJPA pour gérer les transactions de base de données.

Paramètres Utilisés et Valeurs Possibles

`CETAOPEVAL2` :

Utilisation : Code d'état pour les opérations validées.

Valeurs Possibles : Identifiants numériques définis dans le système.

`CETAREMERR` :

Utilisation : Code d'état pour les remises en erreur.

Valeurs Possibles : Identifiants numériques définis dans le système.
