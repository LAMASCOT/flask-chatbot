Introduction

Description : La tache LotCorpReader est conçue pour lire et traiter les fichiers de lots d'opérations bancaires, en extrayant les informations nécessaires et en mettant à jour les enregistrements dans la base de données.

Package

Emplacement : sas.action.readers.corporates

Description Fonctionnelle

Déclenchement : Activée lors de la détection de nouveaux fichiers de lots dans le répertoire configuré.

Traitement :

Lit chaque ligne du fichier pour extraire les informations des opérations bancaires.

Crée des objets Operation à partir des lignes lues.

Met à jour les enregistrements de chèques, virements, effets, et prélèvements dans la base de données si les conditions sont remplies.

Résultat : Les enregistrements sont mis à jour dans la base de données.

En cas d’erreur : Les erreurs de traitement sont consignées dans les journaux pour un suivi ultérieur.

Opérations Clés

Lecture des Fichiers : Utilise un BufferedReader pour lire chaque ligne du fichier et extraire les informations nécessaires.

Création d'Objets :

Utilise la méthode getChamp pour extraire les champs des lignes et créer des objets Operation.

Utilise des expressions régulières pour diviser les lignes en attributs.

Mise à Jour des Enregistrements :

Recherche les enregistrements existants dans la base de données à l'aide de DataBaseJPA.

Met à jour les attributs des enregistrements si les conditions sont remplies.

Met à jour l'état des enregistrements si nécessaire.

Paramètres du Composant et Valeurs Possibles

`VERSION_SICA` : Détermine la version du système SICA utilisée. Les valeurs possibles sont "2" ou "3".

`CETAOPEVALTRANS` : Code d'état pour les opérations en cours de traitement.

`CETAOPEVAL` : Code d'état pour les opérations validées.

`CETAIMASTO` : Code d'état pour les images stockées.

`patternDate` : Format de date utilisé pour le traitement des dates.

`CODE_SICA3` : Code utilisé pour la correspondance des banques dans le système SICA3.

`CODE_CORRESPONDANCE` : Code utilisé pour la correspondance des banques.

Interactions avec la Base de Données

Requêtes SQL : Utilise des requêtes JPQL pour récupérer et mettre à jour les enregistrements de chèques, virements, effets, et prélèvements.

Gestion des Transactions : Utilise DataBaseJPA pour gérer les transactions de base de données, assurant la création et la mise à jour des enregistrements de manière atomique.
