Introduction

Description : La tache Icom2NatWriter est une extension de FlatFileWriter utilisée pour générer et envoyer des fichiers ICOM2 nationaux vers la BCEAO. Elle traite divers types d'opérations financières telles que les chèques, effets, et prélèvements, en fonction de leur état et type.

Package

Emplacement : sas.action.writers.sica

Description Fonctionnelle

Objectif : Générer et envoyer des fichiers ICOM2 nationaux vers la BCEAO en traitant les opérations financières disponibles.

Traitement :

Vérifie les paramètres de compensation manuelle et d'envoi.

Récupère les opérations financières (chèques, effets, prélèvements) d'une banque.

Prépare les opérations en fonction de leur type et destination.

Met à jour l'état des opérations après traitement.

Méthodes

Icom2NatWriter() : Constructeur par défaut qui initialise la description de l'envoi.

execute() : Exécute le processus de génération et d'envoi des fichiers ICOM2.

preparePrelevements(List<Prelevements> prelevements) : Prépare les prélèvements pour l'envoi.

prepareCheques(List<Cheques> cheques) : Prépare les chèques pour l'envoi.

prepareEffets(List<Effets> effets) : Prépare les effets pour l'envoi.

printRejets(EnteteRemise enteteRemise, int i, int j) : Génère une ligne de rejet pour l'impression.

printRemise(DataBaseJPA db) : Imprime la remise et met à jour les états des opérations.

printEnteteLot(EnteteRemise enteteRemise, Remcom remcom, int i) : Génère l'en-tête du lot pour l'impression.

printEnteteRemise(EnteteRemise enteteRemise) : Génère l'en-tête de la remise pour l'impression.

updateEtatOperations(DataBaseJPA db) : Met à jour l'état des opérations après l'envoi.

Paramètres Utilisés

CETAOPEALLICOM2 : État des opérations à traiter pour ICOM2.

CETAOPEALLPREICOM2 : État des opérations avant l'envoi ICOM2.

CETAOPEALLICOM2ENV : État des opérations après l'envoi ICOM2.

COMPENSATION_MANUELLE : Indicateur de compensation manuelle.

DATECOMPENS_NAT : Date de compensation nationale.

VERSION_SICA : Version du système SICA utilisé.

Remarques

Gestion des Erreurs : Les erreurs lors de la génération ou de l'envoi des fichiers sont gérées par des logs et des mises à jour de paramètres.

Extensibilité : La tache peut être étendue pour inclure d'autres types d'opérations ou de traitements spécifiques.

Intégration : Cette tache utilise DataBaseJPA pour interagir avec la base de données et récupérer les opérations à traiter.
