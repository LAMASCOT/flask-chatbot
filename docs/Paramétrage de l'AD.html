PARAMETRAGE DE L’AD

Connexion à l’Active Directory – Mode Simple (LDAP)

Introduction

Ce paramétrage permet de configurer l’authentification des utilisateurs via un annuaire Active Directory en mode standard (LDAP). Il est utilisé dans les institutions bancaires souhaitant centraliser la gestion des identifiants utilisateurs sans recourir à une authentification locale.
Cette configuration est particulièrement adaptée aux environnements d’entreprise où la sécurité, la traçabilité et la gestion unifiée des comptes sont des priorités. Le paramétrage permet de modifier dynamiquement les paramètres de connexion sans intervention dans le code source.

Paramètres de la connexion AD

Nom du paramètre : AD_FOREST_SEARCH
Valeur : BSICBANK.CI
Libellé : Base de Recherche AD
Type : CODE_PARAMS

Description :
Ce paramètre spécifie le domaine Active Directory à utiliser pour les recherches d’utilisateurs. Il s’agit de la base à partir de laquelle les comptes seront interrogés dans l’AD.

Nom du paramètre : AD_CONN_ID
Valeur : SMART
Libellé : Utilisateur de connexion AD
Type : CODE_PARAMS

Description :
Définit l’identifiant de l’utilisateur technique utilisé pour établir la connexion entre l’application et le serveur Active Directory.

Nom du paramètre : AD_CONN_PASSWORD
Valeur : [crypté]
Libellé : Mot de passe user AD
Type : CODE_CRYPTED

Description :
Contient le mot de passe chiffré de l’utilisateur technique (défini par AD_CONN_ID). Ce mot de passe est requis pour s’authentifier auprès de l’Active Directory.

Nom du paramètre : AD_HOST
Valeur : Adresse IP du serveur
Libellé : Adresse IP AD
Type : CODE_PARAMS

Description :
Spécifie l’adresse IP du contrôleur de domaine AD avec lequel l’application doit établir la communication pour l’authentification LDAP.

Nom du paramètre : LDAP_CONNECTION
Valeur : OUI
Libellé : Connexion à un Annuaire pour l'authentification
Type : CODE_PARAMS

Description :
Active ou désactive l’utilisation de l’annuaire AD via le protocole LDAP. Si la valeur est définie à OUI, l’application interrogera l’AD pour authentifier les utilisateurs.

Nom du paramètre : EXTERNAL_USER_CONNECTION
Valeur : OUI / NON / (absent)
Type : CODE_PARAMS

Description :
Contrôle la source d’authentification des utilisateurs :

  • Si OUI : l’authentification passe par une source externe comme l’AD.

  • Si NON : l’authentification est locale, via la base de données de l’application.

  • Si le paramètre est absent : l’application utilise par défaut la base de données locale.

Connexion à l’Active Directory – Mode API avec Token

Introduction

Dans certains environnements, l’authentification via LDAP direct peut être remplacée par une API sécurisée utilisant un système de jetons. Cette méthode repose sur un service intermédiaire qui s’occupe de la validation des identifiants en communiquant avec l’Active Directory via des protocoles modernes d’authentification. Ce mode est utile pour des architectures distribuées ou cloud, ou lorsqu’un accès direct à l’AD n’est pas possible.

Paramètres de la connexion AD via API

Nom du paramètre : AD_AUTH_SERV_TOKEN
Valeur : OUI
Libellé : Connexion AD et TOKEN
Type : CODE_PARAMS

Description :
Active la connexion à l’annuaire Active Directory via une API externe sécurisée par jeton (token). Lorsque ce paramètre est activé, l’application ne passe plus par LDAP mais par un service distant.

Nom du paramètre : AD_AUTH_SERV_TOKEN_TEST
Valeur : OUI
Libellé : Connexion AD et TOKEN test
Type : CODE_PARAMS

Description :
Permet d’activer la connexion AD via jeton pour un environnement de test ou de préproduction. Ce paramètre facilite la validation fonctionnelle avant mise en production.
